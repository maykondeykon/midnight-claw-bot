## Problema: Navegador não detectado

### Erro
```
webbrowser.Error: could not locate runnable browser
```
Script tentou abrir navegador automaticamente mas falhou (ambiente headless/sem browser configurado).

### Solução Implementada
Fluxo OAuth manual para ambiente SSH/headless:
```python
auth_url, _ = flow.authorization_url(prompt='consent')
print("Abra esta URL no navegador:", auth_url)
code = input("Cole o código de autorização aqui: ")
flow.fetch_token(code=code)
```

### Como funciona via SSH
1. Script exibe URL de autorização
2. Usuário abre URL no navegador LOCAL (não precisa ser no servidor)
3. Autoriza acesso no Google
4. Google redireciona para `http://localhost?code=XXXXX` (vai dar erro, normal)
5. Usuário copia o código da URL (parâmetro `code=`)
6. Cola o código no terminal SSH
7. Token é salvo em `~/.config/google/token.json`

### Erro 400: invalid_request
**Erro encontrado:** `flowName=GeneralOAuthFlow`

**Causa provável:** Redirect URI não configurado corretamente ou usuário não está na lista de Test Users.

**Correção aplicada:**
```python
flow.redirect_uri = 'http://localhost'
auth_url, _ = flow.authorization_url(prompt='consent', access_type='offline')
```

**Verificar no Google Cloud Console:**
1. **Google Auth Platform → Audience** → Adicionar email como Test User
2. **Google Auth Platform → Data Access** → Verificar se scopes estão adicionados
3. **Clients** → Verificar se redirect URI `http://localhost` está na lista
