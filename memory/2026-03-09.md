## Discussão: Tipo de Acesso

### Opções Consideradas

| Opção | Acesso | Quando usar |
|-------|--------|-------------|
| **SSH Key** | Total (ler + escrever) | Confiança total |
| **Deploy Key (read-only)** | Apenas ler | Acesso específico por repo |
| **Personal Access Token** | Controlado por scopes | Flexibilidade |
| **GitHub App** | Fino controle | Produção/empresas |

### Se quiser APENAS leitura:
Usar **Deploy Keys** por repositório:
1. Repo → Settings → Deploy keys → Add deploy key
2. Colar chave pública
3. **NÃO marcar** "Allow write access"
4. Salvar

**Vantagem:** Acesso específico, apenas leitura, por repositório.

## PAT (Personal Access Token)

### Fine-grained PAT (Recomendado)
- ✅ Pode selecionar **múltiplos repositórios específicos**
- ✅ Definir acesso apenas leitura
- ✅ Token expira automaticamente
- ✅ Pode revogar a qualquer momento

### Como criar
1. https://github.com/settings/personal-access-tokens/new
2. Token name: `Midnight Claw (Read-only)`
3. Repository access: "Only select repositories" → marcar os repos
4. Permissions → Contents: Read-only, Metadata: Read-only
5. Generate token

### Classic PAT (não recomendado)
- Acesso a TODOS os repositórios ou nenhum
- Menos controle granular

## ⚠️ Limitação de Permissões

### Problema
Tokens fine-grained foram criados sem permissão de **Contents Read-only**.
- ✅ Pode listar repositórios
- ✅ Pode ver metadados
- ❌ **Não pode** ler arquivos/commits

### Solução
Editar tokens em https://github.com/settings/personal-access-tokens e adicionar:
- **Repository permissions → Contents: Read-only**
